Déplacer le contenu statique dans static

Jekyll a une règle : tout dossier ne commençant pas par _ sera copié tel quel dans le dossier de sortie _site. Hugo place tout le contenu statique dans static. Il faut donc tout déplacer là. Avec Jekyll, cela ressemblait à :

▾ <racine>/
    ▾ images/
        logo.png

Cela doit devenir :

▾ <racine>/
    ▾ static/
        ▾ images/
            logo.png

De plus, tous les fichiers devant être à la racine (comme CNAME) doivent être déplacés dans static.

Créer le fichier de configuration Hugo

Hugo peut lire la configuration en JSON, YAML ou TOML. Hugo supporte aussi des paramètres personnalisés. Consulte la documentation de configuration Hugo pour plus de détails.

Définir le dossier de publication sur _site

Par défaut, Jekyll publie dans _site et Hugo dans public. Si, comme moi, tu as _site mappé sur un sous-module git branché sur gh-pages, tu as deux options :

  1. Modifier ton sous-module pour pointer gh-pages vers public au lieu de _site (recommandé).

     git submodule deinit _site
 git rm _site
 git submodule add -b gh-pages git@github.com:ton-utilisateur/ton-repo.git public

  2. Ou, modifier la configuration Hugo pour utiliser _site au lieu de public.

     {
     ..
     "publishdir": "_site",
     ..
 }

Convertir les templates Jekyll en templates Hugo

C’est la plus grosse partie du travail. La documentation est ton amie. Consulte la documentation des templates Jekyll pour te rafraîchir la mémoire sur la construction de ton blog et la documentation des templates Hugo pour apprendre la méthode Hugo.

À titre d’exemple, convertir mes templates pour heyitsalex.net m’a pris seulement quelques heures.

Convertir les plugins Jekyll en shortcodes Hugo

Jekyll a des plugins ; Hugo a des shortcodes. Le portage est assez simple.

Exemple d’implémentation

Par exemple, j’utilisais un plugin Jekyll image_tag personnalisé pour générer des figures avec légende sous Jekyll. En lisant sur les shortcodes, j’ai découvert que Hugo propose un shortcode intégré qui fait exactement la même chose.

Plugin Jekyll :

module Jekyll
  class ImageTag < Liquid::Tag
    @url = nil
    @caption = nil
    @class = nil
    @link = nil
    // Patterns
    IMAGE_URL_WITH_CLASS_AND_CAPTION =
    IMAGE_URL_WITH_CLASS_AND_CAPTION_AND_LINK = /(\w+)(\s+)((https?:\/\/|\/)(\S+))(\s+)"(.*?)"(\s+)->((https?:\/\/|\/)(\S+))(\s*)/i
    IMAGE_URL_WITH_CAPTION = /((https?:\/\/|\/)(\S+))(\s+)"(.*?)"/i
    IMAGE_URL_WITH_CLASS = /(\w+)(\s+)((https?:\/\/|\/)(\S+))/i
    IMAGE_URL = /((https?:\/\/|\/)(\S+))/i
    def initialize(tag_name, markup, tokens)
      super
      if markup =~ IMAGE_URL_WITH_CLASS_AND_CAPTION_AND_LINK
        @class   = $1
        @url     = $3
        @caption = $7
        @link = $9
      elsif markup =~ IMAGE_URL_WITH_CLASS_AND_CAPTION
        @class   = $1
        @url     = $3
        @caption = $7
      elsif markup =~ IMAGE_URL_WITH_CAPTION
        @url     = $1
        @caption = $5
      elsif markup =~ IMAGE_URL_WITH_CLASS
        @class = $1
        @url   = $3
      elsif markup =~ IMAGE_URL
        @url = $1
      end
    end
    def render(context)
      if @class
        source = "<figure class='#{@class}'>"
      else
        source = "<figure>"
      end
      if @link
        source += "<a href=\"#{@link}\">"
      end
      source += "<img src=\"#{@url}\">"
      if @link
        source += "</a>"
      end
      source += "<figcaption>#{@caption}</figcaption>" if @caption
      source += "</figure>"
      source
    end
  end
end
Liquid::Template.register_tag('image', Jekyll::ImageTag)

devient ce shortcode Hugo :

<!-- image -->
<figure {{ with .Get "class" }}class="{{.}}"{{ end }}>
    {{ with .Get "link"}}<a href="{{.}}">{{ end }}
        <img src="{{ .Get "src" }}" {{ if or (.Get "alt") (.Get "caption") }}alt="{{ with .Get "alt"}}{{.}}{{else}}{{ .Get "caption" }}{{ end }}"{{ end }} />
    {{ if .Get "link"}}</a>{{ end }}
    {{ if or (or (.Get "title") (.Get "caption")) (.Get "attr")}}
    <figcaption>{{ if isset .Params "title" }}
        {{ .Get "title" }}{{ end }}
        {{ if or (.Get "caption") (.Get "attr")}}<p>
        {{ .Get "caption" }}
        {{ with .Get "attrlink"}}<a href="{{.}}"> {{ end }}
            {{ .Get "attr" }}
        {{ if .Get "attrlink"}}</a> {{ end }}
        </p> {{ end }}
    </figcaption>
    {{ end }}
</figure>
<!-- image -->

Finitions

Corriger le contenu

Selon le niveau de personnalisation de chaque post Jekyll, cette étape demandera plus ou moins d’efforts. Il n’y a pas de règle stricte, mais hugo server --watch est ton ami. Teste tes changements et corrige les erreurs au besoin.

Nettoyer

Il faut supprimer la configuration Jekyll à ce stade. Supprime tout ce qui n’est plus utilisé.
