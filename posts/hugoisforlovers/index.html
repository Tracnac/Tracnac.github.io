Étape 1. Installer Hugo

Allez sur Hugo releases et téléchargez la version appropriée pour votre OS et architecture.

Sauvegardez-le dans un endroit spécifique car nous allons l’utiliser dans l’étape suivante.

Des instructions plus complètes sont disponibles sur Install Hugo

Étape 2. Construire la documentation

Hugo a son propre site d’exemple qui se trouve être également le site de documentation que vous lisez en ce moment.

Suivez les étapes suivantes :

  1. Clonez le dépôt Hugo
  2. Allez dans le dépôt
  3. Lancez hugo en mode serveur et construisez la documentation
  4. Ouvrez votre navigateur sur http://localhost:1313

Commandes pseudo correspondantes :

git clone https://github.com/spf13/hugo
cd hugo
/chemin/vers/hugo/installé/hugo server --source=./docs
> 29 pages created
> 0 tags index created
> in 27 ms
> Web Server is available at http://localhost:1313
> Press ctrl+c to stop

Une fois arrivé ici, suivez le reste de cette page sur votre build local.

Étape 3. Modifier le site de documentation

Arrêtez le processus Hugo en appuyant sur Ctrl+C.

Maintenant nous allons relancer hugo, mais cette fois avec hugo en mode watch.

/chemin/vers/hugo/de/l/étape/1/hugo server --source=./docs --watch
> 29 pages created
> 0 tags index created
> in 27 ms
> Web Server is available at http://localhost:1313
> Watching for changes in /Users/spf13/Code/hugo/docs/content
> Press ctrl+c to stop

Ouvrez votre éditeur favori et modifiez une des pages de contenu source. Que diriez-vous de modifier ce fichier même pour corriger la faute de frappe. Que diriez-vous de modifier ce fichier même pour corriger la faute de frappe.

Les fichiers de contenu se trouvent dans docs/content/. Sauf indication contraire, les fichiers sont situés au même emplacement relatif que l’URL, dans notre cas docs/content/overview/quickstart.md.

Modifiez et sauvegardez ce fichier… Remarquez ce qui s’est passé dans votre terminal.

> Change detected, rebuilding site

> 29 pages created
> 0 tags index created
> in 26 ms

Rafraîchissez le navigateur et observez que la faute de frappe est maintenant corrigée.

Remarquez comme c’était rapide. Essayez de rafraîchir le site avant qu’il ait fini de se construire. Je vous mets au défi. Avoir un retour quasi instantané vous permet de laisser votre créativité s’exprimer sans attendre de longs builds.
